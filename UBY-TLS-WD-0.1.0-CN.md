## 1. 摘要

UBY（宇宙大爆炸年）是一种用于跨尺度时间标注的约定性时间坐标系。它以宇宙学模型中的共动时间原点作为约定零点，以标准儒略年作为基本单位，将宇宙历史、地质历史、人类历史及长远未来事件置于一条单调递增的标注轴上。

UBY 不旨在取代现有时间系统，而是提供一种统一、可溯源、版本化的辅助时间标签，用于跨尺度可视化、长期归档、数据索引、科普叙事、跨学科数据交换，以及大规模跨域数据整合。

UBY 的显著特征是 **双轨制设计**：
- **语义轨**：保留原始时间的全部要素（单位、数值、不确定性、测年模型、精度等级）；
- **数值投影轨**：将每个事件投影到以大爆炸为锚点的单一实数轴上。

语义轨保留原始时间表示的保真度；数值投影轨使跨域邻近查询成为可能——当各学科只保留各自原始单位时，这种查询是不可行的。

UBY 不得被解释为与模型无关的绝对物理时间。一个 UBY 值的含义取决于：

1. 所使用的 UBY 规范版本；
2. 所使用的宇宙学模型或锚点版本；
3. 原始时间数据的来源与精度；
4. 用例对应的精度等级；
5. 取整规则与不确定性声明。

## 2. 范围

### 2.1 适用范围

本规范适用于以下场景：

1. 跨越宇宙历史、地质历史、人类历史的统一时间标注；
2. 长持续时间数据档案的辅助时间索引；
3. 与原始时间字段共存的科学数据集补充标识符；
4. 博物馆、教育、科普、数字人文中的连续时间叙事；
5. 需同时引用日历日期、儒略日、地质年代、红移或宇宙年龄的跨学科文档；
6. 万年以上记录、文明档案、超长期说明材料的辅助时间锚点；
7. 软件系统、数据库、可视化、API 中的跨尺度时间标签；
8. **需要跨异构时间单位进行邻近 JOIN 的大规模跨学科数据整合**；
9. **需要单一可排序数值轴跨越十几个数量级时间的可复现数据挖掘管线**。

### 2.2 不适用范围

本规范不得用于以下场景：

1. 法律文件、合同、金融结算、行政审批等依赖法定日期时间的场景；
2. 卫星导航、时间传递、航天测控、深空导航等高精度工程场景；
3. 替代原子时、协调世界时、GNSS 时等专业授时系统；
4. 作为天文、地质、考古原始数据的唯一权威时间记录；
5. 需要相对论原时、坐标时或轨道动力学的工程计算；
6. 将大爆炸前时间或宇宙奇点解释为直接测量的物理时间；
7. 任何因时间信息误用可能造成法律、安全、金融或工程风险的场景。

### 2.3 与已有时间系统的关系

UBY 是补充性标注层。正式数据必须保留原始时间表示，例如：

- UTC / TAI；
- ISO 8601；
- JD / MJD；
- 地质年代或地层单位；
- BP、ka、Ma、Ga；
- 红移 z；
- 原始测年报告及关联的不确定性。

UBY 可以作为辅助字段加入，但不得删除或替代原始时间字段。

## 4. 术语与定义

### 4.1 UBY
UBY（宇宙大爆炸年）是一种连续时间标注系统，以宇宙学模型中的共动时间原点为约定零点，以标准儒略年为单位。UBY 值表示自约定零点以来经过的年数。

### 4.2 UBY 表达式
符合本规范语法的 UBY 文本表示，例如：

```text
UBY 13787002026 [model=LCDM-Planck2018] [spec=0.1.0]
```

### 4.3 UBY 值
UBY 表达式中的数值部分，以标准儒略年表示。

### 4.4 标准儒略年
```text
1 儒略年 = 365.25 × 86400 s = 31557600 s
```
UBY 的基本单位为标准儒略年。它是常量定义，不随地球自转或轨道的长期变化而改变。

### 4.5 宇宙学共动时间
FLRW 宇宙学模型中，随宇宙平均膨胀运动的理想观测者所测得的时间坐标。在 UBY 中，共动时间仅作为建模标注坐标，不构成与模型无关的绝对时间。

### 4.6 模型版本
标识 UBY 值所依赖的宇宙学模型、参数来源、参数历元、锚点或修订信息。推荐格式：

```text
[model=<模型族>-<来源或参数集>[-<修订标识>]]
```

### 4.7 规范版本
标识 UBY 表达式所遵循的语法、字段与解释规则。例：`[spec=0.1.0]`。

### 4.8 锚点
用于将 UBY 值与原始时间系统对齐的约定参考点。锚点通常包含：锚点标识符、原始时间、JD 值、UBY 值、模型版本、规范版本。

### 4.9 精度等级
描述 UBY 值的可靠性来源与适用范围。本规范定义三级：
1. Level 1：近地相对计量级；
2. Level 2：跨尺度比例叙事级；
3. Level 3：宇宙学模型相关级。

### 4.10 不确定性
表示 UBY 值的误差范围、置信水平或模型依赖程度。

### 4.11 语义轨（Semantic Track）
UBY 记录中保留原始时间表示的字段集合：`original_time_unit`、`original_time_value`、`original_error`、`uby_model`、`uby_precision_level`、`uby_precision_label`。任何符合本规范的统一时间轴数据集必须保留语义轨。

### 4.12 数值投影轨（Numeric Projection Track）
通过 §11 算法将原始时间投影到 UBY 轴得到的单一实数字段 `uby_value`。数值投影轨使跨域邻近 JOIN（§19）与大规模数值排序成为可能。

### 4.13 双轨制设计
UBY 的设计原则：每条记录同时携带语义轨（§4.11）与数值投影轨（§4.12）。两轨不可互换：语义轨保保真度，数值投影轨保可计算性。规范定义见 §5.7。

### 4.14 跨域 JOIN
一种数据库操作，检索两个或多个学科中 `uby_value` 满足邻近谓词（通常 `|a.uby_value − b.uby_value| < τ`）的记录对。当各学科仅保留原始单位时，跨域 JOIN 不可行。

### 4.15 跨域信号零假设检验
一种统计过程，将观测到的跨域对齐与随机化零模型进行比较。规范定义见 §20。

## 5. 设计原则

### 5.1 补充性原则
UBY 必须作为原始时间系统的补充字段使用，不得取代 UTC、TAI、JD、地质年代、红移或原始测年结果。

### 5.2 模型透明性原则
任何依赖宇宙学模型、参数集或锚点假设的 UBY 值，必须声明模型版本或可溯源的锚点信息。

### 5.3 无精度膨胀原则
UBY 输出不得暗示高于原始材料或模型所支持的精度。若来源仅支持百万年级精度，UBY 表达式不得显示未支持的小数年、日级或秒级精度。

### 5.4 版本可溯源原则
正式数据交换与长期归档中的 UBY 表达式应声明规范版本。推荐形式：`[spec=0.1.0]`。

### 5.5 可计算互操作性原则
实现者必须能够从锚点、单位、取整规则、模型版本与规范版本复现 UBY 值。

### 5.6 误用禁止原则
实现、文档与用户界面不得将 UBY 描述为：绝对时间；权威授时系统；原始科学数据的替代品；法律或金融时间依据；高精度工程时间系统。

### 5.7 双轨制设计原则

声称符合本规范、整合多于一个学科的 UBY 数据集，必须为每条记录同时携带两轨：

1. **语义轨**——原始时间字段（单位、数值、不确定性）加 UBY 精度元数据（模型、精度等级、精度标签）。这些字段保留来源保真度，不得由数值投影轨派生或覆盖。
2. **数值投影轨**——通过 §11 算法由语义轨得到的单一实数字段 `uby_value`。此字段是用于跨域邻近 JOIN（§19）、排序与跨学科数值比较的**唯一**字段。

双轨制不是冗余表示：每轨承担独立、不可替代的角色。
- 删除语义轨会破坏保真度（`uby_value=135715499995.6` 无法反演出 `±0.4 Ma` 不确定性或 `地层学` 测年方法）；
- 删除数值投影轨会破坏跨域可 JOIN 性。

实现可将双轨以分层 API 暴露：语义轨为结构化对象，数值投影轨为标量索引列。两层必须由同一底层记录支撑。

仅携带 `uby_value` 而丢弃原始字段的数据集，自 0.1.0 起不符合本规范。

### 5.8 跨域可证伪性原则

任何基于 UBY 对齐提出的跨域时间信号主张，必须附带 §20 规定的零假设检验。未通过零假设检验的观测对齐必须被报告为"不显著"，不得被描述为发现。

## 6. 精度等级

### 6.1 概览

| 等级 | 名称 | 主要范围 | 可靠性来源 | 典型用途 | 限制 |
| --- | --- | --- | --- | --- | --- |
| Level 1 | 近地相对计量级 | 公元前后约 ±1,000,000 年 | 原始历史、天文、考古、地质或测年数据 | BCE/CE 跨界连续编号、排序、索引 | 不提高来源精度 |
| Level 2 | 跨尺度比例叙事级 | 宇宙、地球与生命演化大尺度事件 | 来源版本、模型近似、有效数字 | 跨尺度展示、科普、长期归档 | 不适用于高精密度量计算 |
| Level 3 | 宇宙学模型相关级 | 高红移、早期宇宙、模型积分事件 | 宇宙学模型与参数集 | 辅助宇宙学标注与模型内比较 | 不同模型的值不可直接比较 |

### 6.2 Level 1：近地相对计量级

适用场景：原始时间来源为公历、ISO 8601、JD、历史纪年或近地测年证据。

Level 1 主要提供 BCE/CE 边界的连续编号与索引。实现者必须注意：
1. UBY 不修正原始历史来源中的不确定性；
2. BCE 年必须明确区分传统 BC 编号与 ISO 天文年编号；
3. 若原始日期不精确，输出不得显示虚假高精度；
4. Level 1 可省略模型标签，但正式数据仍应保留锚点或模型信息。

### 6.3 Level 2：跨尺度比例叙事级

适用场景：太阳系形成、地质时间区间、生命演化、大灭绝、长远未来事件等大尺度事件。

Level 2 输出应使用恰当的有效数字显示。例：

```text
UBY 9.22G [model=LCDM-Planck2018] [spec=0.1.0]
```

不应写为：

```text
UBY 9220000000.000000 [model=LCDM-Planck2018] [spec=0.1.0]
```

除非原始来源支持该精度。

### 6.4 Level 3：宇宙学模型相关级

适用场景：CMB 退耦、第一代恒星形成、高红移星系、由红移积分得到的宇宙年龄等。

Level 3 表达式必须声明模型版本。例：

```text
UBY 380K [model=LCDM-Planck2018] [spec=0.1.0]
```

不同模型版本下的 Level 3 UBY 值不得直接比较，除非提供了显式转换方法或重算结果。

## 7. 记法

### 7.1 总体结构

```text
UBY <值> [model=<模型标识>] [spec=<规范版本>]
```

- `UBY` 为固定前缀；
- `<值>` 为 UBY 值；
- `[model=...]` 为模型版本标签；
- `[spec=...]` 为规范版本标签。

### 7.2 完整数值格式

用于计算、存储、数据库与正式交换。语法：

```text
UBY <十进制数> [model=<模型标识>] [spec=<规范版本>]
```

示例：

```text
UBY 13787002026 [model=LCDM-Planck2018] [spec=0.1.0]
UBY 13786999780 [model=LCDM-Planck2018] [spec=0.1.0]
UBY 380000 [model=LCDM-Planck2018] [spec=0.1.0]
```

规则：
1. 不得使用千位分隔符；
2. 小数位数不得超过来源材料支持的精度；
3. 正式交换应包含 `[spec=...]`；
4. Level 2 与 Level 3 应包含 `[model=...]`；
5. 若上下文已声明锚点，Level 1 可省略 `[model=...]`。

### 7.3 量级简写格式

用于显示、图表与科普材料。语法：

```text
UBY <数><量级符号> [model=<模型标识>] [spec=<规范版本>]
```

量级符号：

| 符号 | 含义 | 倍数 |
| --- | --- | --- |
| K | 千年 | 10³ |
| M | 百万年 | 10⁶ |
| G | 十亿年 | 10⁹ |
| T | 万亿年 | 10¹² |

示例：

```text
UBY 380K [model=LCDM-Planck2018] [spec=0.1.0]
UBY 9.22G [model=LCDM-Planck2018] [spec=0.1.0]
UBY 13.787G [model=LCDM-Planck2018] [spec=0.1.0]
```

量级简写格式不得作为唯一长期存储格式。

### 7.4 科学计数法格式

```text
UBY <系数>×10^<指数> [model=<模型标识>] [spec=<规范版本>]
```

示例：`UBY 3.8×10^5 [model=LCDM-Planck2018] [spec=0.1.0]`。纯 ASCII 环境可用 `x` 替代 `×`。

### 7.5 学术助记格式

用于人类可读的 Level 1 显示。语法：

```text
UBY <前缀><符号><年6位> [model=<模型标识>] [spec=<规范版本>]
```

- `<前缀>` 为 6 位助记前缀；
- `<符号>` 为 `+` 或 `-`；
- `<年6位>` 为天文年的 6 位绝对值。

助记前缀计算规则：`前缀 = ROUND(t0_Ga × 10000)`，其中 `t0_Ga` 为模型给出的当前宇宙年龄（十亿年）。

Planck 2018 示例：`t0_Ga = 13.787`，`前缀 = ROUND(13.787 × 10000) = 137870`。

示例：

```text
UBY 137870+002026 [model=LCDM-Planck2018] [spec=0.1.0]
UBY 137870-000220 [model=LCDM-Planck2018] [spec=0.1.0]
```

与完整数值的关系：`UBY <P>+YYYYYY = UBY (P × 100000 + YYYYYY)`。

助记格式不得用于正式计算。

### 7.6 公众友好助记格式

```text
UBY <前缀> AD<年> [model=<模型标识>] [spec=<规范版本>]
UBY <前缀> BC<年> [model=<模型标识>] [spec=<规范版本>]
```

示例：

```text
UBY 137870 AD2026 [model=LCDM-Planck2018] [spec=0.1.0]
UBY 137870 BC221 [model=LCDM-Planck2018] [spec=0.1.0]
```

正式数据交换不应仅依赖公众友好助记格式。

## 8. 规范版本标签

### 8.1 语法

```text
[spec=<语义化版本>]
```

示例：`[spec=0.1.0]`、`[spec=1.0.0]`、`[spec=1.0.0-rc.1]`。

### 8.2 规则

1. `[spec=...]` 表示 UBY 表达式所遵循的规范版本；
2. `[spec=...]` 不表示宇宙学模型版本；
3. 模型版本必须通过 `[model=...]` 或元数据字段 `model_version` 表示；
4. 正式数据交换与长期归档应包含 `[spec=...]`；
5. 若内联标签与外部元数据冲突，内联标签优先，并应记录警告；
6. 若 `[spec=...]` 缺失且上下文未声明版本，解析器不得自动假设最新版本。

## 10. 基准锚点

### 10.1 默认锚点

本规范 0.1.0 定义的默认锚点如下：

```text
锚点 ID: UBY-ANCHOR-2026-01-01Z
原始时间: 2026-01-01T00:00:00Z
JD: 2461041.5
UBY: 13787002026.0
模型: LCDM-Planck2018
规范: 0.1.0
```

规范表达：

```text
2026-01-01T00:00:00Z
= JD 2461041.5
= UBY 13787002026.0 [model=LCDM-Planck2018] [spec=0.1.0]
```

### 10.2 锚点解释

此锚点是工程转换的约定锚点，不暗示宇宙年龄具有年级精度。构造逻辑：

```text
Planck 2018 近似当前宇宙年龄: 13.787 Ga
13.787 Ga = 13787000000 儒略年
对应 CE 2026 的天文年: 2026
默认年级锚点: 13787000000 + 2026 = 13787002026
```

### 10.3 锚点变更

若未来规范或模型采用新锚点，必须遵循 §25 的迁移协议。

## 11. 转换算法

### 11.1 JD 转 UBY

在默认锚点下，Level 1 的 JD → UBY 转换：

```text
UBY = (JD - 2461041.5) / 365.25 + 13787002026.0
```

### 11.2 UBY 转 JD

```text
JD = (UBY - 13787002026.0) × 365.25 + 2461041.5
```

### 11.3 ISO 8601 转 UBY

ISO 8601 时间戳应先转换为 JD，再用 §11.1 转换为 UBY。实现者应声明所用时间尺度（UTC、TAI 或 TT）。

### 11.4 天文年级 UBY

仅需年级标注时，可用：

```text
UBY_year = 13787000000 + 天文年
```

### 11.5 BCE/CE 转 UBY

对于传统 BC 年 `n`（1 BC 为 CE 1 的前一年）：

```text
天文年 = 1 - n
UBY_year = 13787000000 + (1 - n)
```

示例：221 BC → 天文年 -220 → `UBY 13786999780 [spec=0.1.0]`。

### 11.6 BP / ka BP / Ma BP / Ga BP 转 UBY

对于以"距今年数"表示的值 `t_bp`，其中"今"定义为锚点年（默认 2026）：

```text
UBY = 13787002026.0 - t_bp_儒略年数
```

其中 `t_bp_儒略年数` 为 BP 值转换为儒略年数。ka BP 乘以 1000；Ma BP 乘以 10^6；Ga BP 乘以 10^9。

### 11.7 红移转 UBY（Level 3）

对于红移 `z`，对应 UBY 由 `z` 处的宇宙学共动年龄计算：

```text
UBY(z) = z=0 处宇宙年龄 - z 处宇宙年龄 = t0 - t(z)
```

其中 `t(z)` 为宇宙学模型给出的 `z` 处共动年龄。必须声明模型版本。不同宇宙学模型计算的 UBY 值不得直接比较。

### 11.8 十进制历年转 UBY

对于十进制历年 `y`（如 2026.5）：

```text
UBY = 13787002026.0 + (y - 2026.0)
```

## 12. 取整与有效数字

### 12.1 取整规则

默认取整规则：Level 1 用年向下取整，Level 2/3 用有效数字舍入。

### 12.2 有效数字

UBY 值的有效数字位数不得超过来源材料支持的位数。由 ±0.4 Ma 不确定性来源派生的值，不得携带超过 7 位有效数字。

### 12.3 显示与存储

存储的 `uby_value` 应保留转换的全部数值精度。显示格式（§7.2–7.6）可截断或舍入以提高可读性。

## 18. 安全、误用与风险

### 18.1 法律与工程风险
UBY 不得用作法定时间、合同时间、金融结算时间或工程控制时间。

### 18.2 科学误用风险
UBY 不得替代原始科学源数据。地质、考古、天文与宇宙学数据必须保留原始测量、不确定性与来源。

### 18.3 伪精度风险
以高精度十进制形式书写大尺度或高不确定事件是误导性的。实现者与数据发布者必须避免伪精度。

### 18.4 模型混用风险
不同宇宙学模型下的 UBY 值可能包含系统性差异。跨模型比较必须声明映射或重算值。

### 18.5 国际化风险
正式机器可读字段应优先使用 ASCII。人名、解释性文本与展示层可以使用 Unicode。

推荐作者署名：

```text
Han Bo
```

### 18.6 跨域伪相关风险
跨域邻近 JOIN（§19）可能产生统计上预期而非物理有意义的对齐。任何作为信号报告的跨域对齐必须附带 §20 规定的零假设检验。实现与出版物在缺少此类检验时不得将跨域对齐报告为发现。

## 19. 跨域 JOIN 方法论

### 19.1 动机

当两个学科使用不兼容的时间单位（如古生物学的 `Ma BP` 与天文学的 `decimal_year`）时，若没有统一轴，则其记录之间的邻近 JOIN 不可行。UBY 数值投影轨提供了这样的轴。

### 19.2 定义

两个记录集 A 与 B 之间的**跨域邻近 JOIN** 是满足以下条件的记录对 `(a, b)` 集合：

```text
|a.uby_value - b.uby_value| < τ
```

其中 `τ` 为以儒略年表示的邻近阈值。

### 19.3 参考 SQL

参考 SQL 形式：

```sql
SELECT a.event_name, b.event_name, ABS(a.uby_value - b.uby_value) AS delta
FROM uby_events a
JOIN uby_events b
  ON a.event_category = :cat_a
 AND b.event_category = :cat_b
 AND ABS(a.uby_value - b.uby_value) < :tau
ORDER BY delta;
```

### 19.4 阈值选择

阈值 `τ` 应参考两侧精度等级中较粗的一方：

- Level 1 × Level 1：`τ` 量级为年；
- Level 1 × Level 2：`τ` 量级为 10^5 年；
- Level 2 × Level 2：`τ` 量级为 10^5–10^6 年；
- Level 3 × 任意：比较需要显式模型声明，`τ` 量级为 10^6 年或更大。

实现应在结果元数据中记录所选 `τ` 与两侧精度等级。

### 19.5 必需输出字段

每个跨域 JOIN 结果集必须包含：

- `a.event_name`、`b.event_name`；
- `a.uby_value`、`b.uby_value`；
- `delta = ABS(a.uby_value - b.uby_value)`；
- `a.event_category`、`b.event_category`；
- `a.uby_precision_level`、`b.uby_precision_level`；
- `tau`（所用阈值）；
- `null_test_result`（见 §20）：`not_tested` / `significant` / `not_significant` 之一。

### 19.6 局限

跨域 JOIN 确立的是**时间邻近性**，而非**因果关系**。满足邻近谓词的记录对 `(a, b)` 并不暗示 `a` 导致 `b`，也不暗示两者共享物理机制。任何因果主张需要独立的物理证据。

## 20. 跨域信号零假设检验

### 20.1 动机

跨域邻近 JOIN 通常会产生非零对齐计数，即使两个记录集在统计上独立。这是因为两个集合常常集中在同一时间窗口（如现代天文记录与现代火山记录都集中在 1990–2025）。需要零假设检验来区分信号与采样偏差。

### 20.2 参考过程

参考零假设检验为**蒙特卡洛标签置换检验**，协议如下：

1. 设 `T_a = {a_i.uby_value}` 与 `T_b = {b_j.uby_value}` 为两个记录集的 UBY 值集合。
2. 设 `N_obs = |{(i, j) : |a_i - b_j| < τ}|` 为观测跨域对齐计数。
3. 在零假设 `H0`（A/B 标签可交换）下，将合并集 `T = T_a ∪ T_b` 汇总后，随机将 `|T_a|` 个值分配给集合 A、`|T_b|` 个值分配给集合 B。此操作保留合并集的时间集中度，同时打破 A-B 区别。

   > **原理说明。** 若从 `T_b` 自身的经验分布中有放回地重采样（自助法），对齐计数统计量会出现退化：由线性性可知 `E[N_mc] = N_obs`，故 z 分数恒为零，检验永无法拒绝。标签置换零模型避免了该退化：它将合并值在两个集合间重新分配，从而产生有意义的零分布——只要存在真实共址，其均值严格小于 `N_obs`。

4. 对 `M` 次蒙特卡洛迭代（推荐 `M ≥ 1000`），计算跨域对齐计数 `N_mc`。
5. 计算零分布均值 `μ` 与标准差 `σ`。
6. 计算 z 分数：`z = (N_obs - μ) / σ`。
7. 计算经验 p 值：`p = (1 + |{m : N_mc[m] >= N_obs}|) / (M + 1)`。

### 20.3 显著性决策

| 条件 | 决策 |
| --- | --- |
| `z < 0` | 观测对齐**低于随机预期**。不显著。 |
| `0 <= z < 2` | 观测对齐**在随机预期内**。不显著。 |
| `2 <= z < 3` | **弱信号**。谨慎报告；需多重检验校正。 |
| `z >= 3` 且 `p < 0.01` | **显著信号**。可作为候选发现报告，但须经物理机制验证。 |

### 20.4 多重检验校正

若同一数据集与 `K` 个不同跨域对象进行检验，必须对每检验显著性阈值进行 Bonferroni 或 Benjamini-Hochberg 校正。

### 20.5 物理机制验证

统计显著的跨域对齐在缺少独立、文档化的、合理连接两类事件的物理机制前，不得报告为科学发现。统计显著性是必要而非充分条件。

### 20.6 参考实现

本检验的参考实现必须暴露：

- 观测对齐计数 `N_obs`；
- 零分布 `(μ, σ)`；
- z 分数；
- 经验 p 值；
- 蒙特卡洛迭代次数 `M`；
- 所选阈值 `τ`；
- 两个记录集的精度等级。

### 21.3 异常值检测

`uby_value` 违反物理边界的记录必须被隔离。隔离日志必须记录：

- `event_id`；
- `original_time_value`；
- 计算的 `uby_value`；
- 违反的边界；
- 采取的行动（`dropped` / `flagged` / `corrected`）。

### 21.4 完整性审计

符合规范的数据集必须报告：

- 每个 `event_category` 的总记录数；
- 缺失各语义轨字段的记录数；
- 缺失各溯源字段的记录数；
- 被物理边界过滤器标记的记录数。

### 21.5 可复现性审计

符合规范的数据集必须附带 `metadata.json` 边车，记录：

- UBY 规范版本；
- 模型版本；
- 锚点标识符；
- 构建时间戳（UTC）；
- 来源数据集列表及每来源记录数；
- §21.4 的 QC 摘要；
- 生成数据集的软件版本。

## 22. 溯源与审计

### 22.1 记录级溯源

符合规范的统一时间轴数据集中每条记录必须携带：

- `source_dataset`：来源数据集名称；
- `source_record_id`：在来源数据集内唯一标识该记录的稳定标识符；
- `source_doi`（应该）：来源数据集 DOI；
- `source_record_uri`（应该）：来源记录的可解析 URI；
- `attribution`（应该）：来源许可证要求的署名字符串。

### 22.2 数据集级溯源

符合规范的数据集必须附带 `PROVENANCE.md`（或等效机器可读）文档，记录：

- 每个来源数据集的名称、版本、发布日期、DOI、许可证；
- 用于生成统一数据集的转换管线；
- 所用软件版本；
- 构建时间戳；
- §21 的 QC 摘要。

### 22.3 FAIR 对齐

符合规范的数据集应与 FAIR 原则对齐：

- **可发现**：数据集及其来源的持久标识符（DOI）；
- **可访问**：通过认可仓库（Zenodo、Figshare、Dryad）开放访问；
- **可互操作**：§13.2 双轨 schema 支持跨学科 JOIN；
- **可重用**：每记录保留每来源许可证与署名。

### 22.4 审计日志

面向长期归档的数据集应维护审计日志，记录每次重建、每次 QC 过滤行动、每次锚点迁移（见 §25）。

## 23. 技术验证框架

### 23.1 范围

本节定义符合规范的 UBY 数据集必须进行并记录的最低技术验证。

### 23.2 转换验证

对每个转换路径（§11.1–11.8），测试套件必须验证：

- 往返同一性：`convert_back(convert_forward(x)) ≈ x`（在声明精度内）；
- 附录 C 的最小测试向量；
- Level 3 路径的模型依赖声明。

### 23.3 跨域 JOIN 验证

符合规范的数据集必须包含至少一个跨域 JOIN（§19）的工作示例，连同零假设检验结果（§20）。该示例必须可从发布的数据集与代码复现。

### 23.4 采样偏差验证

对于任何报告为显著（§20.3）的跨域信号，验证必须包含：

- 检查信号非共享时间集中度的伪迹（如两个记录集都集中在 1990–2025）；
- 阈值 `τ` 的敏感性分析；
- 若检验多于一个跨域对象，须做多重检验校正。

### 23.5 重建可复现性

从相同来源与软件版本的两次独立重建必须产生字节相同的 `uby_value` 字段（允许小于 1 ULP 的浮点表示差异）。

## 24. 与已有工作的关系

### 24.1 Cosmic Calendar（Sagan, 1977）

Cosmic Calendar 将宇宙历史映射到单个日历年用于科普可视化。UBY 共享以大爆炸为零点的概念选择，但有三个区别：

1. UBY 是可查询的数值轴，非可视化隐喻；
2. UBY 携带双轨 schema（§13）保留原始时间字段；
3. UBY 定义正式转换规范（§11）与一致性等级（§15）。

### 24.2 Unified Time Framework（UTF, Wang et al., 2022）

UTF 提出基于本体的地学时间框架，含六类时间元素（time type / expression / reference / unit / uncertainty / dating method），以 RDF/OWL 节点组织，通过 SPARQL 查询。UBY 与 UTF 采用**互补的设计哲学**：

| 维度 | UTF | UBY |
| --- | --- | --- |
| 设计哲学 | 基于本体；保留语义多样性 | 双轨；保留语义多样性**并**追加数值投影 |
| 数据模型 | RDF 三元组 + 6 元素本体 | 关系型；语义轨列 + 单一 `uby_value` 列 |
| 查询语言 | SPARQL | SQL |
| 数值投影 | 不提供 | 提供（`uby_value`） |
| 跨域 JOIN | 不直接支持 | 支持（§19） |
| 零假设检验 | 不提供 | 提供（§20） |
| 已发布数据集 | 未发布 | 参考实现附 C2 符合规范的统一时间轴数据集 |
| 学科覆盖 | 地学（地质、地理、古生物） | 宇宙学、天文学、地质学、古生物学、古气候、古生态、火山学、 instrumental、spaceflight |
| 时间跨度 | 显生宙 + 人类历史（约 5×10^8 年） | 17 个数量级（大爆炸至今） |

UBY 不声称在"跨学科统一时间"概念上对 UTF 具有概念新颖性。UBY 相对 UTF 的贡献为：

1. 将统一时间概念实例化为 150 万+ 条跨学科数据库；
2. 将时间轴锚定到物理参考点（大爆炸 = 0）而非抽象根节点；
3. 追加数值投影轨与跨域 JOIN 方法论（§19）；
4. 追加跨域信号零假设检验协议（§20）。

### 24.3 Universal Space-Time Referencing System（USTRS）

USTRS 提出用于导航与深空应用的宇宙时空编码。UBY 共享宇宙学原点的选择，但范围不同：USTRS 面向导航的时空定位，UBY 面向跨学科数据整合。两者不直接可比。

### 24.4 ISO 8601、JD、ICS

UBY 是与这些既有系统互操作的补充层（见 §16），不替代它们。

### 24.5 定位声明

UBY 宜被理解为基于本体的时间框架（如 UTF）的**物理坐标补充**：它保留 UTF 提供的语义保真度，同时追加使跨域邻近查询在大规模下可行的数值投影。UBY 适用于大规模跨学科数据整合与基于邻近的数据挖掘；UTF 仍适用于知识图谱构建与语义推理。

## 25. 锚点版本迁移协议

### 25.1 何时需要迁移

在以下情况下需要锚点迁移：

1. 宇宙学模型参数变更（如 Planck 2018 → Planck 2025）；
2. 锚点参考历元变更（如 2026-01-01 → 2030-01-01）；
3. 儒略年定义修订（不太可能；将为 MAJOR 版本变更）。

### 25.2 迁移记录

迁移必须以以下字段记录：

| 字段 | 说明 |
| --- | --- |
| `migration_id` | 迁移唯一标识符 |
| `old_anchor_id` | 旧锚点标识符 |
| `new_anchor_id` | 新锚点标识符 |
| `old_uby_value` | 旧锚点下样本 UBY 值 |
| `new_uby_value` | 新锚点下样本 UBY 值 |
| `offset` | `new_uby_value - old_uby_value`（对所有记录为常数） |
| `affected_precision_levels` | {1, 2, 3} 的子集 |
| `migration_date` | UTC 时间戳 |
| `migration_software_version` | 执行迁移的软件版本 |
| `migration_notes` | 自由文本注释 |

### 25.3 迁移过程

1. 冻结旧数据集并归档。
2. 计算新旧锚点间的常数偏移量。
3. 对每条记录，设 `new_uby_value = old_uby_value + offset`。
4. 按 §7 更新 `uby_value_text` 与 `uby_mnemonic_iso`。
5. 更新 `anchor_id`、`model_version`、`spec_version` 字段。
6. 对迁移后数据集运行 §21 QC 检查。
7. 按 §25.2 发布迁移记录。

### 25.4 兼容性

若仅锚点参考历元变更（如 2026 → 2030），Level 1 记录受常数偏移影响；Level 2/3 记录是否受影响取决于模型参数是否同时变更。迁移记录必须声明受影响的精度等级。

### 25.5 向后兼容性

在旧锚点下构建的数据集对归档使用仍然有效，但在不同锚点下构建的记录之间的跨数据集 JOIN 不得在未迁移的情况下进行。实现应通过 `anchor_id` 字段检测锚点不匹配，并拒绝 JOIN 或触发迁移。

## 26. 增量更新机制

### 26.1 何时允许增量更新

增量更新（追加新记录而不重建整个数据集）在以下情况下允许：

1. 锚点未变更（见 §25）；
2. 模型版本未变更；
3. 新记录符合相同 schema（§13.2）；
4. 新记录已经 §21 QC 检查。

### 26.2 更新过程

1. 按 schema（§13.2）与 QC 边界（§21.2）验证新记录。
2. 以单调递增的 `event_id` 追加新记录。
3. 更新数据集 `metadata.json`：
   - 新总记录数；
   - 新来源数据集列表（若新增来源）；
   - 更新时间戳；
   - 执行更新的软件版本。
4. 对样本重新运行跨域 JOIN 验证（§23.3）以确认无回归。

### 26.3 何时需要全量重建

在以下情况下需要全量重建：

- 锚点变更（见 §25）；
- 模型版本变更；
- 规范版本引入不兼容 schema 变更；
- QC 过滤器追溯应用于既有记录。

### 26.4 增量更新的版本化

每次增量更新应分配形如 `<数据集版本>+<更新序号>` 的构建标识符，如 `v0.1.0+001`。构建标识符必须记录于 `metadata.json`。
